# EXP-001: Data Audit + Corpus Manifest + Split Freeze

**Status:** Planned
**Wave:** 1 (Foundation)
**Depends on:** Nothing
**Blocks:** EXP-002, EXP-003, EXP-004

---

## Goal

Validate all data artifacts, build the corpus manifest, create the frozen dev/test split, and run the capacity audit needed for Doc-to-LoRA packaging decisions.

## Specification

### 1. Corpus Manifest
- Create `data/manifests/corpus_manifest.csv` with columns:
  - `doc_id` (filename hash)
  - `filename`
  - `page_count`
  - `approx_token_count` (via tokenizer or heuristic)
  - `source_type` (statute/case/cross_case if identifiable)
- Verify all 65 PDFs are readable and extractable

### 2. Goldset Validation
- Load `dev-gold-150-v1.benchmark.json`
- Validate schema: all 150 entries have question_id, answer, answer_type, gold_retrieval
- Verify all doc_ids in gold_retrieval exist in corpus manifest
- Check for near-duplicate / paraphrase questions (embedding similarity > 0.95)
- Report answer_type distribution, difficulty distribution, tag distribution

### 3. Split Creation
- Create stratified 120/30 dev/test split
- Stratify by: answer_type, difficulty, single/multi-doc
- Ensure date (7) and hard (6) strata have at least 1-2 items in test
- Group any near-duplicates before splitting
- Save to `data/splits/split_v1.json` with question_ids per split
- Freeze: this split must not change after creation

### 4. Capacity Audit
- Total corpus token count (sum across all docs)
- Distribution of document lengths (pages, tokens)
- How many Doc-to-LoRA segments needed to cover full corpus
- How many segments per cluster (rough estimate at k=4)
- Summary: is monolithic single-pass feasible? Expected answer: no

### Target files
- `data/manifests/corpus_manifest.csv`
- `data/splits/split_v1.json`
- `experiments/EXP-001_data_audit/REPORT.md`
- `src/data/io.py` (corpus loader, goldset loader)
- `src/data/preprocessing.py` (tokenization, page extraction)
- `src/data/splits.py` (split generation)

### Acceptance criteria
- [ ] corpus_manifest.csv exists with 65 rows, no missing fields
- [ ] All gold_retrieval doc_ids exist in manifest
- [ ] split_v1.json exists with 120 dev + 30 test question_ids
- [ ] Stratification is reasonable (no empty strata in test)
- [ ] Capacity audit numbers documented in REPORT.md
- [ ] No near-duplicate leakage across split boundary

**Result:** Done
